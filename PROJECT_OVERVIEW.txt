================================================================================
                      END-TO-END FLOW
================================================================================

1. INDEXING (One-time, on first server startup)
-----------------------------------------------
   PDF Document (neet.pdf)
         ↓
   pypdf extracts text page-by-page (83 pages)
         ↓
   LlamaIndex creates Document objects with metadata:
   - file_name: "neet.pdf"
   - page_label: "1", "2", ... "83"
         ↓
   OpenAI Embeddings API converts text chunks → 1536-dimension vectors
   (API: text-embedding-ada-002)
         ↓
   Vectors stored in SimpleVectorStore
         ↓
   Persisted to ./storage/ folder as JSON files


2. QUERY FLOW (Per question)
----------------------------
   User types question in frontend (localhost:3000)
         ↓
   Frontend sends POST /chat request to backend (localhost:8000)
         ↓
   Backend converts question → vector using OpenAI Embeddings API
         ↓
   Vector similarity search (cosine similarity) → retrieves Top 3 relevant chunks
         ↓
   Retrieved chunks + question combined into prompt
         ↓
   Sent to GPT-4o-mini for answer generation
         ↓
   LLM generates answer based ONLY on retrieved context
         ↓
   Response sent back with:
   - Answer text
   - Source citations (file name + page numbers)
         ↓
   Frontend displays answer with "Verified from Official Document" badge

================================================================================
                      COST ESTIMATION
================================================================================

ONE-TIME INDEXING COST (83 pages, ~50K tokens):
- Embeddings: ~50,000 tokens × $0.0001/1K = ~$0.005 (half a cent)

PER QUERY COST BREAKDOWN:
-------------------------------------------------
Component              | Tokens  | Cost
-------------------------------------------------
Question embedding     | ~20     | $0.000002
Context (3 chunks)     | ~1,500  | $0.000225 (input)
LLM response           | ~200    | $0.0006 (output)
-------------------------------------------------
TOTAL PER QUERY        |         | ~$0.001 (0.1 cent)
-------------------------------------------------

COST SUMMARY:
- 1,000 queries   ≈ $1
- 10,000 queries  ≈ $10
- 100,000 queries ≈ $100

Pricing based on:
- GPT-4o-mini: $0.15/1M input tokens, $0.60/1M output tokens
- Embeddings: $0.10/1M tokens


================================================================================
                      VECTOR DATABASE DETAILS
================================================================================

Type: SimpleVectorStore (LlamaIndex built-in)

CHARACTERISTICS:
- In-memory vector store with disk persistence
- Storage location: ./storage/ directory (JSON files)
- Search algorithm: Brute-force cosine similarity
- Performance: Fast for <100K vectors
- No external database required

WHY THIS CHOICE:
- Perfect for demos and small-medium documents
- Zero configuration needed
- Data persists across server restarts
- Easy to understand and explain

FOR PRODUCTION SCALE, consider:
- Pinecone (managed, serverless)
- Weaviate (open-source)
- Qdrant (open-source)
- Chroma (embedded)
- PostgreSQL + pgvector

================================================================================
                      COMMON QUESTIONS & ANSWERS
================================================================================

Q: What happens if the answer is not in the document?
A: The AI responds with "This information is not available in the document."

Q: Can we add more documents?
A: Yes! Just add PDF files to backend/data/ folder and restart the server.
   The index will be rebuilt automatically.

Q: How do we update the NEET bulletin for next year?
A: Replace neet.pdf in backend/data/, delete the storage/ folder, and 
   restart the server to rebuild the index.

Q: Is the data stored on OpenAI servers?
A: The document text is sent to OpenAI for embedding only. OpenAI's API 
   doesn't train on customer data.

Q: Can this work offline?
A: No, it requires internet connection for OpenAI API calls. For offline,
   you'd need to use local models like Llama or Mistral.

Q: How accurate is the similarity search?
A: We retrieve top 3 most relevant chunks using cosine similarity. The
   threshold is tuned for high precision in Q&A scenarios.
